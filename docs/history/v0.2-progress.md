## What landed this session (nested compositor)

- **Nested Wayland compositor end-to-end** (`crates/lmux-wayland-host`): smithay 0.7 on a dedicated OS thread, binds `wl_compositor` / `wl_shm` / `wl_seat` / `xdg_wm_base` under `$XDG_RUNTIME_DIR/lmux-<pid>`, emits `HostEvent::{Ready,ToplevelCreated,ToplevelClosed,FrameReady,Stopped,…}` on an `async_channel::Receiver`, and accepts `HostCommand::{SetGeometry,SendKey,SendPointer,Shutdown,…}` on a `Sender`. `HostHandle` drops → shutdown + thread join. Wire-level tests: `bind_socket.rs` (socket + Ready/Stopped) and `xdg_toplevel.rs` (real wayland-client handshake, set_title/set_app_id, ToplevelCreated/Closed assertions). SHM buffer commit → `FrameReady` covered by `shm_frame.rs`; xkb + pointer/keyboard forwarding by `input.rs`.
- **`SatelliteWidget`** (`crates/lmux/src/satellite.rs`): `gtk::Picture` wrapped in a `Frame`, fed by `HostEvent::FrameReady` → `gdk::MemoryTexture`. Uses `add_tick_callback` for size tracking (gtk4's `Picture` has no `connect_resize`), `EventControllerKey/Motion/Scroll/Focus` + `GestureClick` translate back to `HostCommand::SendKey` / `SendPointer` (XKB keycodes = Linux scancode + 8). Carries `PaneId`, `surface_id`, `title`, `app_id`, `closed: Rc<Cell<bool>>`.
- **Pane enum refactor** (`crates/lmux/src/pane.rs`): previous `Pane` struct renamed → `TerminalPane`; new `pub enum Pane { Terminal(TerminalPane), Satellite(SatelliteWidget) }` with delegating methods (`id`, `widget`, `grab_focus`, `has_exited`, `terminate`, `kill`) plus terminal-only `Option`-returning helpers (`cwd`, `child_pid`, `snapshot_cwd`, `snapshot_thumbnail`, `cell_size`) and satellite-noop setters (`set_font`, `set_bell_callback`, `attach_controllers`). Call-sites in `state.rs` kept working unchanged.
- **Host-event dispatcher in AppState** (`crates/lmux/src/state.rs`): new fields `wayland_host: Option<HostHandle>`, `host_cmd_tx: Option<Sender<HostCommand>>`, `surface_to_pane: HashMap<SurfaceId, PaneId>`, `wayland_display_name: Option<String>`. `install_wayland_host` parks the handle and sender; `handle_host_event` dispatches all 7 variants. `on_toplevel_created` allocates a fresh `PaneId` (`next_id.saturating_add(1)`), builds a `SatelliteWidget`, splices the pane into the focused workspace via `layout.replace_leaf` (`Dir::Vertical`, ratio 0.5), registers it in `panes` / `pane_uuids` / `surface_to_pane` / `pane_workspace`, and rebuilds the widget tree. `on_toplevel_closed` marks the widget closed and removes the leaf unless it is the last pane.
- **GTK-side wiring** (`crates/lmux/src/app.rs`): `install_wayland_host(&state)` after config-watch. Pulls `lmux_wayland_host::start()`, registers the handle on AppState, and drives `evt_rx.recv().await` from `glib::MainContext::default().spawn_local` with a `Rc::downgrade(state)` weak ref (no strong-ref cycle). Graceful degradation: failure just logs a warn and returns — cockpit still boots as a pure-terminal multiplexer.
- **Launcher redirect** (`crates/lmux/src/launcher.rs` + `crates/lmux-compositor/src/spawn.rs`): new `spawn_tagged_with_env(argv, cwd, wayland_display: Option<&str>)` forces `WAYLAND_DISPLAY=<nested>` + `GDK_BACKEND=wayland` + `QT_QPA_PLATFORM=wayland` and strips `DISPLAY` when the nested socket exists, so a Wayland-capable toolkit cannot fall back to X11. `spawn_tagged_with_pid` delegates (pass-through with `None`). Launcher reads `state.wayland_display_name()` and routes through the new entry point.
- **Env override tests** (`crates/lmux-compositor/tests/spawn_env.rs`, 2 cases): positive — `WAYLAND_DISPLAY`/`GDK_BACKEND`/`QT_QPA_PLATFORM` pinned, `DISPLAY` stripped, `LMUX_SATELLITE_ID` preserved (regression guard for v0.1 contract). Negative — `None` passes parent env verbatim.
- **Clippy cleanup**: `#![allow(clippy::expect_used, clippy::unwrap_used)]` on new test files; targeted `#[allow]` on two pre-existing `.expect()` sites in `lmux-wayland-host/src/state.rs` (`seat.add_keyboard`, `client_compositor_state`).

Tasks closed: ADR-0018 #12 (pane-tree integration), #13 (launcher redirect), #14 (smoke tests — spawn_env + wire-level xdg/shm/input). Full GTK-display e2e deferred as manual QA (the cockpit won't launch a nested GTK app under cargo test without a display server).

## What landed this session (continued)

- **GUI launcher + hotkey (`Ctrl+B l`) + sidebar button** (`crates/lmux/src/launcher.rs`): freedesktop `.desktop` scanner + Exec parser + substring-filtered modal Window dialog + spawn via `spawn_tagged_with_pid`. Replaced initial Popover with a modal `Window` because Popovers parented to ApplicationWindow landed offscreen on KWin/Wayland.
- **KWin placement of spawned satellites**: `KwinCompositor::spawn_satellite` now returns immediately with (uuid, pid) and schedules a tokio task that injects a one-shot `lmux-place-<pid>` script via the scripting D-Bus; the snippet snaps the matching window into the right half of its placement area.
- **Anchor-tied satellite lifecycle**: every satellite spawned from the launcher is registered under the active anchor (`AppState::register_satellite`). `set_active_anchor` broadcasts `SetSatelliteVisible { pid, visible }` over a new `compositor_bridge` (dedicated thread + tokio current_thread runtime + `async_channel`), and `KwinCompositor::set_window_visible_by_pid` toggles `w.minimized` via a one-shot KWin script. Net effect: GUI satellites hide on anchor switch-away and re-appear on switch-back, mirroring terminal-pane workspace membership.
- **E2E tests** for `lmux-cli satellite open` (invalid UUID rejected, argv required, bus error when no cockpit, nil UUID default accepted, help lists `open`) — `crates/lmux-e2e/tests/satellite_open.rs`.
- **Unit tests** for `.desktop` scan + Exec parse + override resolution (5 new tests in `launcher.rs`).
